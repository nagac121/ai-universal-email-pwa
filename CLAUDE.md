# Claude Code Environment Guide for AI Universal Email PWA

## Build & Development
- **Development server**: `npm run dev` (runs `next dev --webpack`)
- **Production build**: `npm run build` (runs `next build --webpack`)
- **Start built app**: `npm start` (runs `next start`)
- The `--webpack` flag forces Next.js 16 to use the legacy Webpack bundler, required for our custom PWA setup.

## Testing
- **Run tests**: `npm test` *(alias for `jest` – see `package.json` scripts)*
- Tests are written in TypeScript using **Jest** and **@testing-library/react**.

## Code Style Guidelines
- **Language**: TypeScript – enable `strict` mode, no `any`.
- **React / Next.js 16**:
  - Use functional components with hooks.
  - Follow the file‑based routing conventions of Next.js 16.
  - Export pages with `export default function Page()`.
- **Accessibility (a11y)**:
  - All form fields must have associated `<label>` elements or `aria-label`.
  - Use `react-hook-form` for validation and error handling.
  - Ensure interactive elements are keyboard‑focusable.
- **List Rendering**:
  - Provide stable, unique `key` props (e.g., IDs, not array index).
  - Avoid using mutable indices for keys.
- **Styling**: Tailwind CSS – keep classes composable; lint with `eslint` and `eslint-config-next`.
- **Linting**: Run `npm run lint` to enforce the above rules.
